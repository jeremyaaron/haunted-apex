# Haunted Apex v0.5.0 Entanglements Clarifications

These decisions lock the open questions from [`v0.5.md`](./v0.5.md). The v0.5 system
should be thin, legible, and deterministic where possible. Contacts should feel like a
new strategic surface, not a hidden narrative trap.

## Universal Options

Every active, non-burned contact has both universal options.

### Cultivate

```ts
{
  kind: 'cultivate',
  label: 'Cultivate',
  cost: {
    resources: 600,
  },
  contactEffects: {
    trust: 10,
    volatility: -6,
    exposure: 2,
  },
}
```

Player meaning:

```text
Spend time and money to make the contact safer, more cooperative, and less volatile.
```

### Pressure

```ts
{
  kind: 'pressure',
  label: 'Pressure',
  effects: {
    intel: 4,
    ruin: 2,
  },
  contactEffects: {
    leverage: 10,
    trust: -6,
    volatility: 8,
  },
}
```

Player meaning:

```text
Extract immediate value by making the relationship more coercive and unstable.
```

## Manage Contact

`Manage Contact` does not allow operative assignment in v0.5.

It should work like `Work the Ledger`:

```text
action + target + option
```

Not:

```text
action + operative + target + option
```

## Contact Status

Contact status is derived in selectors, not persisted.

Persist:

```ts
type ContactState = {
  id: ContactId;
  trust: number;
  leverage: number;
  volatility: number;
  exposure: number;
  burned: boolean;
  recentInteractions: ContactInteraction[];
  flags: Record<string, boolean | number | string>;
};
```

Derive:

```ts
function deriveContactStatus(contact: ContactState): ContactStatus {
  if (contact.burned) return 'burned';
  if (contact.volatility >= 75) return 'volatile';
  if (contact.trust >= 70 && contact.leverage < 50) return 'trusted';
  if (contact.leverage >= 65 && contact.trust < 50) return 'pressured';
  if (contact.trust >= 50 && contact.leverage >= 50) return 'entangled';
  if (contact.trust < 25 && contact.leverage < 25) return 'cold';
  return 'useful';
}
```

## Burned Contacts

A burned contact remains visible but is no longer usable.

Mechanical rules:

```text
Burned contacts remain in the Contacts panel.
Burned contacts keep their history.
Burned contacts cannot be targeted by Manage Contact.
Burned contacts cannot provide services.
Burned contacts are excluded from normal contact event eligibility.
Burned contacts remain in final reports.
Burned contacts do not recover in v0.5.
```

UI treatment:

```text
Muted card
Status: Burned
Services disabled
Recent history still visible
```

## Ledger Effects

Contact service Ledger effects are explicit and previewable.

Prefer deterministic Ledger effects in v0.5:

```text
Creates Debt: Private Investor Terms
Creates Secret: Dead Channel Trace
Creates Favor: Checkpoint Captain
```

Do not use hidden "may create" effects in the first implementation.

If a Ledger entry has `relatedContactId`, using or resolving that entry modifies the
contact only through declared contact effects. The contact relationship and the contact
consequence are separate concepts:

```text
relatedContactId creates the relationship.
contactEffects define the consequence.
```

## Inactive Contacts

Inactive contacts are invisible in normal UI for v0.5.

The player sees three active contacts. The remaining definitions exist only in the
content registry and seeded selection pool. They may appear in debug output or simulation
reports if useful.

## Quiet Treatment

Include `Quiet Treatment` in v0.5.

It automatically targets the highest-stress current roster operative. It does not add an
operative selector and does not make `Manage Contact` operative-assignable.

Preview example:

```text
Quiet Treatment

Cost:
Resources -1100

Expected:
Heat -4
Juno Hex Stress -10

Dr. Mercy:
Exposure +3
```

Selection rule:

```ts
function selectQuietTreatmentTarget(state: GameState): OperativeId | undefined {
  return currentRosterOperatives(state)
    .filter((operative) => operative.stress > 0)
    .sort((a, b) => b.stress - a.stress)[0]?.id;
}
```

Disable `Quiet Treatment` when all current roster operatives have `0` Stress.

## Contact Event Recurrence

Signature contact events fire at most once per run:

```text
Veyra's Room
Hollis Is Being Watched
Mercy's Bill
Ciro's Route Remembers
Confession Leak
```

Generic contact events may repeat, but:

```text
Generic contact events cannot repeat within 2 weeks.
After firing once, their weight is reduced by 50% for the rest of the run.
```

## Role Tags

Add both `social` and `stability` to `ContactRoleTag`.

```ts
type ContactRoleTag =
  | 'heat_control'
  | 'intel'
  | 'loyalty'
  | 'resources'
  | 'ruin'
  | 'ledger'
  | 'rival_pressure'
  | 'nightlife'
  | 'security'
  | 'weird'
  | 'social'
  | 'stability';
```

Use `social` for Veyra / Mina style contacts.

Use `stability` for Dr. Mercy / Hollis / recovery-oriented contacts.
