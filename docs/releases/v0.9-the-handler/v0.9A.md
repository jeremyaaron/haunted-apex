# Haunted Apex v0.9.0 — The Handler Clarifications

## 1. Training Run seed policy

Use **one deterministic curated Training config** for v0.9.

Do not start with a generated safe pool. That adds uncertainty before the Handler system is proven.

Locked rule:

```text
Training Run uses one fixed curated seed/config in v0.9.
```

Suggested shape:

```ts
const TRAINING_RUN_CONFIG = {
  runMode: 'training',
  seed: 'TRAINING-GLASS-CROWN-001',
  campaignTensionId: 'dirty_capital',
  advisorMode: 'handler',
  dominionTarget: 80,
};
```

Why Dirty Capital works well:

```text
It gives the player early resources.
It teaches fronts/factions/debts without starting in immediate crisis.
It gives HandlerBot room to demonstrate stabilization and pacing.
```

If Dirty Capital proves too obligation-heavy for first play, tune the Training config rather than adding more Training seeds.

## 2. Standard tuning obligation

Yes. This should be explicit.

If HandlerBot loses any Standard validation run:

```text
500 Standard validation runs attempted
500 HandlerBot wins required
0 HandlerBot losses allowed
```

then v0.9 is not done.

The response should be:

```text
1. Fix HandlerBot blind spots if the logic is clearly wrong.
2. Tune campaign/event/action numbers if the bot is making sane choices and still losing.
3. Lower Dominion or soften events only if the failure pattern shows Standard is too tight.
```

Locked rule:

```text
v0.9 should not release until HandlerBot wins 500/500 Standard validation runs or the gate is intentionally revised based on evidence.
```

This is the core quality bar.

---

## 3. Dominion targets

Yes.

Use:

```text
Training Dominion Target: 80
Standard Dominion Target: 90
```

Do not lower Standard immediately. First test HandlerBot.

Locked rule:

```text
Training starts at 80. Standard remains 90 unless Handler validation proves Standard tuning is too tight.
```

If Standard fails validation primarily by Dominion shortfall, then the first likely fixes are:

```text
slightly increase Dominion yields
soften event drag
improve Handler pacing logic
then consider lowering Standard target
```

Lowering Standard Dominion should be a balance decision, not the first move.

## 4. Default Advisor mode

Use this behavior:

```text
Training Run:
Handler by default.

Standard Run:
Use saved player preference if present.
If no preference exists, default to Coach.
```

Why Coach instead of Hints:

```text
Hints may be too vague for a first-time Standard player.
Handler may feel too prescriptive outside Training.
Coach gives direction while preserving decision-making.
```

Locked rule:

```ts
function getDefaultAdvisorMode(runMode, savedPreference) {
  if (runMode === 'training') return 'handler';
  return savedPreference ?? 'coach';
}
```

Yes. Label custom Standard seeds as **Unvalidated** unless runtime Handler validation exists.

Player-facing copy:

```text
Custom Seed: Unvalidated
This seed has not been checked against Handler guidance and may be harder or less balanced than standard generated runs.
```

For v0.9:

```text
Training fixed seed = Validated
Standard generated test seeds = harness-validated, not necessarily individually certified at runtime
Custom seed = Unvalidated
```

Locked rule:

```text
Custom seeds are marked Unvalidated in v0.9 unless explicitly passed through validation.
```

## 8. Recommendation highlighting

Yes. Highlighting is allowed and recommended.

No auto-apply is locked, but highlighting is not automation.

In Handler mode, highlight:

```text
recommended action card
recommended target option
recommended operative option
recommended event choice
```

Highlight style should be clear but not overwhelming:

```text
Recommended by Handler
```

or

```text
Handler Pick
```

Locked rule:

```text
Handler recommendations may visually highlight controls, but the player must click them manually.
```

This gives training-wheel clarity without taking agency away.

## 9. Handler recommendations after partially queued orders

Yes. Handler must adapt to partially queued orders.

This is important.

Behavior:

```text
If no orders queued:
  show full recommended plan.

If one order queued:
  evaluate current queued order.
  recommend best remaining order if command remains.

If two orders queued:
  evaluate current queued plan.
  show either “Plan looks stable” or warnings/suggested replacements.

If queued order is risky:
  explain the risk and suggest removing/replacing it.
```

Example:

```text
You queued Run Small Job at Zero Mercy.

Handler Warning:
This raises Heat and provokes Knox. If you keep it, use your second Command to reduce Heat or build recovery.

Recommended remaining order:
Bribe an Official
```

Locked rule:

```text
Advisor recommendations are stateful and adapt to the current queue.
```

This prevents the Advisor from fighting the player.

## 10. Validation sample size

Use deterministic validation sets.

For v0.9, lock this:

## Training

Because Training uses one fixed deterministic config:

```text
Replay the fixed Training config with HandlerBot.
It must win.
```

Running it 100 times is fine as a regression check, but if the engine is deterministic, it is the same test repeated. The meaningful test is:

```text
The fixed Training trace is stable and wins.
```

Acceptance:

```text
Training fixed config: Handler win required.
```

## Standard

Use a fixed deterministic validation set:

```text
100 seeds per campaign tension
5 campaign tensions
500 Standard validation runs
```

The validation set should be reproducibly generated or committed as static test data.
It is a build-level confidence gate, not a certified playable seed bank.

Example:

```text
standard-validation-seeds
Corp Crackdown: 100 seeds
Nightlife War: 100 seeds
Ghostline Signal: 100 seeds
Industrial Cut: 100 seeds
Dirty Capital: 100 seeds
```

Acceptance:

```text
HandlerBot wins 500/500 Standard validation runs.
Any HandlerBot loss blocks release until investigated.
No invalid recommendations.
No softlocks.
```

Optional extra:

```text
Run an additional random smoke batch after the fixed bank, but do not use it as the formal gate.
```

Locked rule:

```text
v0.9 release gate uses a fixed 100-seed-per-campaign Standard validation set.
```
